SQL PERFORMANCE — INDEX GUIDE

เขียน Query แบบไหน
ให้ระบบ ใช้ Index ได้

Index เปรียบเหมือนสารบัญในหนังสือ — ถ้าเขียน Query ถูกวิธี ระบบกระโดดไปยังข้อมูลได้ทันที แต่ถ้าผิด ต้องพลิกทุกหน้าตั้งแต่ต้น

RULE 01 อย่าครอบคอลัมน์ด้วยฟังก์ชัน
orders.sql
ช้า — Index ช่วยไม่ได้
WHERE YEAR(create_date) = 2026
เร็ว — ใช้ Index ได้เต็มที่
WHERE create_date >= '2026-01-01'
  AND create_date <  '2027-01-01'
🔴
ปัญหา — Function Wrap
ฟังก์ชัน YEAR() ถูกเรียกใช้กับ ทุกแถว ก่อนจะเปรียบค่า ระบบมองไม่เห็น Index จึงต้องสแกนตารางทั้งหมดตั้งแต่ต้น
🟢

RULE 02 ระวัง % ข้างหน้าใน LIKE
employees.sql
ช้า — Full Table Scan
WHERE name LIKE '%สมชาย'
เร็ว — Index Prefix Seek
WHERE name LIKE 'สมชาย%'
🔴
ปัญหา — Wildcard ข้างหน้า
%สมชาย แปลว่า "ลงท้ายด้วยสมชาย" ระบบไม่รู้ว่าจะเริ่มหาที่ไหน เหมือนหาคนที่ "นามสกุลขึ้นต้น" โดยไม่รู้นามสกุล — ต้องพลิกสมุดโทรศัพท์ทั้งเล่ม
🟢
วิธีแก้ — Prefix ข้างหน้า
สมชาย% Index รู้ prefix — กระโดดไปหมวด "ส" แล้วอ่านแค่ entries ที่ขึ้นต้นด้วย "สมชาย" หยุดเมื่อเจอตัวอักษรอื่น
